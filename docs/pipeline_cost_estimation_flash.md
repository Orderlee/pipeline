## 7. 비용 최적화 팁

| 방법 | 절감 효과 | 설명 |
|------|----------|------|
| 짧은 영상(< 5초) 필터링 | -10~15% | 이벤트 감지 가치가 낮은 영상 제외 |
| Context Caching 활용 | Input -90% | 동일 프롬프트 반복 시 캐시 적용 ($0.03/M) |
| 배치 크기 증가 (YOLO) | 시간 -10% | batch=8로 변경 |
| Gemini 동시 호출 수 증가 | 시간 -40% | RPM 한도 내 최대화 |
| 2-pass 전략 | 비용 -50% | Flash로 1차 스크리닝 → Pro로 고가치 영상만 재처리 |

### Context Caching 적용 시 비용 (503건)

| 항목 | 일반 | 캐시 적용 | 절감 |
|------|------|----------|------|
| Input 비용 | $0.50 | $0.05 | -90% |
| Output 비용 | $0.91 | $0.91 | 0% |
| **총 비용** | **$1.41** | **$0.96** | **-32%** |

> Context Caching은 동일한 시스템 프롬프트를 반복 사용할 때 적용 가능합니다.
> 파이프라인에서 `VIDEO_EVENT_PROMPT`가 모든 영상에 동일하게 사용되므로 캐시 효과가 큽니다.
